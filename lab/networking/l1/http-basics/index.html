Networking · Level 1 · 30–45m

Capture & Replay Lab

Practice intercepting HTTP traffic with Burp Suite, replaying modified requests, and documenting observations for your lab report.

Requirements

  • Browser + Burp Proxy
  • Kali VM (optional)
  • HTTPBin account (no login required)

Environment

  • External link: PortSwigger labs
  • Credentials: none
  • Notes: Use HTTP history to compare requests

Goal

Capture a GET request to httpbin.org, replay it with Burp Repeater after altering headers, and document the impact on the response payload.

Tasks

  1. Configure Burp Suite to intercept traffic from your browser and verify certificates are trusted.
  2. Send a request to https://httpbin.org/anything and forward it to Burp Repeater.
  3. Modify the User-Agent and X-Demo headers, send the request, and compare responses.
  4. Document differences and annotate screenshots for your submission.

Hints

If Burp isn’t intercepting, confirm your browser proxy settings point to 127.0.0.1:8080 and Burp’s intercept is on.

Use the Repeater "Headers" tab to add a custom header. Watch the JSON response field "headers" for your changes.

Deliverables

  • Screenshot showing the modified request and response in Burp Repeater.
  • Short explanation (2–4 lines) of how the server reflected the header changes.
  • Request/response snippet pasted into your report template.

Safety

Stay within provided labs. Do not proxy real production traffic or sensitive accounts.
