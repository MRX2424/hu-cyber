HTTP Basics

Build intuition for requests, responses, methods, status codes, headers, and cookies so you can reason about every interaction captured in Burp or DevTools.

Beginner Theory + Demo

Objectives

  • Describe the structure of HTTP requests and responses.
  • Differentiate between GET and POST behaviors.
  • Explain the purpose of key headers like Host, Cookie, and User-Agent.
  • Capture and replay a request using browser DevTools.

Overview

HTTP is a text-based protocol that transfers documents, APIs, and assets across the web. Every request includes a method, target, version, headers, and optional body. Servers respond with status codes, headers, and the content body. When you intercept traffic with Burp Suite or DevTools, you read and manipulate these components to validate behavior and uncover weaknesses such as improper caching, insecure cookies, or parameter trust issues.

Demo steps

  1. Open Firefox and press F12 to launch DevTools. Select the Network tab and enable "Persist Logs".
  2. Visit https://httpbin.org/get and observe the captured GET request. Note the Request URL, User-Agent, and Accept headers.
  3. Right-click the request → Copy → Copy as cURL. Paste into a terminal to replay. Modify the -H "User-Agent" value and rerun.
  4. Discuss what changed in the response, highlighting how headers influence server behavior.
